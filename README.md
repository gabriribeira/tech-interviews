# Frontend Code Review Interview Exercises

This repository contains **code review exercises** used during technical interviews for **Frontend Software Engineer** positions.

Each exercise simulates a **pull request opened by a teammate** in a typical product development workflow. The candidate is asked to review the changes, identify potential issues, and suggest improvements.

The focus of the interview is not on writing new code but on demonstrating the ability to **analyze existing code critically**, communicate feedback clearly, and reason about engineering trade-offs.

---

# Purpose

Code reviews are a core part of everyday software development. They require a combination of skills:

* Understanding unfamiliar code quickly
* Identifying bugs and edge cases
* Evaluating architecture and maintainability
* Spotting performance or scalability issues
* Ensuring code quality and consistency
* Communicating feedback constructively

These exercises are designed to evaluate those abilities in a controlled and time-boxed scenario.

---

# How the Exercises Work

Each exercise represents a **simulated pull request** and typically contains:

* A **pull request title**
* A **short description of the feature or change**
* A **small set of modified files**
* A minimal project structure for context

Candidates are asked to review the changes as if they were part of the team.

Typical prompts during the interview:

* What feedback would you leave on this pull request?
* Are there any potential bugs or edge cases?
* Are there improvements you would suggest?
* Is there anything that could cause performance or maintainability issues?

Follow-up questions may explore how the candidate would **fix or redesign specific parts of the implementation**.

---

# Technology Stack

The exercises use a simplified frontend stack common in modern web applications:

* React
* TypeScript
* Functional components
* React hooks
* Basic browser APIs

The goal is not to test framework trivia but to evaluate **general frontend engineering practices**.

---

# What Interviewers Look For

During the review, interviewers typically assess:

### Code Quality

* Naming and readability
* Type safety
* Separation of concerns

### React Knowledge

* Correct hook usage
* Component structure
* Rendering behaviour

### Async and Data Handling

* Race conditions
* Error handling
* Request lifecycle

### Performance Awareness

* Unnecessary renders
* Expensive computations
* Inefficient data handling

### Product and UX Awareness

* Loading states
* Error states
* Accessibility considerations

### Communication

* Ability to explain issues clearly
* Prioritization of feedback
* Constructive suggestions

# Notes
