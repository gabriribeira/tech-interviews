# Code Review Exercise — Event Search

## Context

This project represents a simplified feature from a sports betting platform.

The page displays a list of sports events and allows users to search for events by team name.

A teammate has opened a pull request implementing the **search functionality** for the events list. The PR description is in [`PULL_REQUEST.md`](./PULL_REQUEST.md) — start there to understand what the developer intended to ship.

Your task is to review the code as if this pull request had been opened in your team.

---

## Task

Perform a **code review** of the changes in this project.

Assume the feature works functionally, but the implementation may contain issues or areas for improvement.

During the review, consider questions such as:

* Are there potential bugs or edge cases?
* Are React hooks used correctly?
* Are there performance concerns?
* Is the code maintainable and well structured?
* Are there missing UX states?
* Are there accessibility concerns?

You are encouraged to explain **what you would change and why**.

---

## Notes

The codebase is intentionally small and simplified so the focus remains on reasoning and code quality rather than project complexity.

The implementation may contain intentional issues to encourage discussion during the review.
